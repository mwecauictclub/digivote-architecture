# Contributing to This Documentation

This repository contains the public architecture documentation for MWECAU DigiVote.

## Goals

- Keep documentation accurate and up-to-date with the running system.
- Make it easy for MWECAU IT staff and future maintainers to understand the high-level design.
- Remain suitable for public sharing (no secrets, no implementation internals that should stay private).

## How to Contribute

1. Propose changes via issues or direct edits.
2. Prefer small, focused pull requests.
3. Update both the relevant `.md` file and any affected Mermaid diagrams.
4. Diagrams live in `diagrams/` as standalone `.mmd` files.

## Style

- Use clear, professional language suitable for university IT departments.
- Include Mermaid diagrams wherever a visual helps.
- Keep sensitive operational details (exact IPs, secrets, internal procedures) out of this repo.

Thank you for helping keep the public record of the system accurate.
